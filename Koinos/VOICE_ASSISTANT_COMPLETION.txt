 ARCHITECTURE OVERVIEW
 ════════════════════════════════════════════════════════════════════════════

 Data Flow:

 User Input
    ↓
 VoiceAssistantView
    ↓
 VoiceAssistantViewModel.sendMessage()
    ↓
 KoinosVoiceRepository.processVoiceCommand()
    ↓
 Get Firebase Token from AuthRepository
    ↓
 POST /api/voice/ with Bearer Token
    ↓
 FastAPI Backend (LangGraph Agent)
    ↓
 AI Response
    ↓
 Parse JSON Response
    ↓
 Add to Messages Array
    ↓
 Update ViewModel State
    ↓
 SwiftUI Re-renders (automatic)
    ↓
 User Sees New Message


 Security Flow:

 1. User sends message → VoiceAssistantView
 2. ViewModel calls Repository
 3. Repository gets Firebase token from AuthRepository
 4. Token injected into Bearer header automatically
 5. HTTP POST to backend with token
 6. Backend receives request with Bearer token
 7. Backend verifies token with Firebase Admin SDK
 8. Backend extracts verified user_id from token
 9. Backend processes message with verified user identity
 10. Response returned to iOS app
 11. Message displayed in chat with user identity intact
